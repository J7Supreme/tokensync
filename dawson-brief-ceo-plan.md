## Vision

### 10x Check
A PM wakes up, checks their phone, and a push notification says "2 things need your
attention this morning." They tap it — a segment-aware brief generated at 6am from
overnight reports, risks in red, one tap to call a portfolio company CEO. No login,
no dashboard, no new tab. Context finds them. The morning routine is replaced, not
supplemented.

### Platonic Ideal
A single screen that looks like it was prepared by a brilliant analyst who knows your
exact portfolio, read everything overnight, filtered for your segment, and wrote it in
your language. You read it in 90 seconds. You close it and start working.

The standalone dashboard is the demo wedge. The real product is a brief-as-workflow
(email/Slack-first delivery), with the dashboard as a "read more" destination.

## Key CEO Insight
The PE dashboard graveyard is full of tools that required "another login." The tools
that survive integrate into existing workflow. This demo is correctly scoped as a
standalone page for demo purposes — but the production roadmap must prioritize
workflow-native delivery (email at 7:30am, Slack DM) from day one. Don't let the
demo shape the production architecture.

## Demo Architecture Clarification (resolves apparent contradictions)

**"Segment personalization without login":** There is no authentication in the demo.
Segment selection is explicit — a visible persona switcher button (Alex / Sarah) at
the top of the page. The user selects their persona. No auth, no session, no DB.
This is a demo, not a product.

**"Demo" vs. "Production vision":**
- DEMO (what ships): Single static HTML page. Persona switcher. Pre-baked brief data.
  Presenter-triggered alert. GitHub Pages.
- VISION (what we pitch): Email delivery, real data pipeline, multi-firm platform.
  The product vision page shows this — it is not a working feature, it is a pitch mockup.

**Toast notification trigger:** Keyboard shortcut `Shift+A` fires the alert during demo.
Copy: "HIGH ALERT: Meridian LP anchor withdrew — deal at risk. Review by 2pm."
Fires once per page load. Must be tested on presenter's actual hardware before demo day.

**Data source:** All brief content is hardcoded JSON in a single JS file. No API calls,
no backend, no dynamic generation. The "intelligence" is written by hand and validated
by a PE expert before demo. Authenticity comes from language quality, not live data.
