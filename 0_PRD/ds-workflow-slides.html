token.json

Semantic

04 — Skills by Use Case

Two use cases, different goals

UC 01

Generate or Clean Up Design System

Build the system from scratch, from a messy existing one, or from code. Goal: all bindings ready, all semantic meanings scoped — so designers can keep working in Figma.

/figma-generate-library · /audit-design-system · /apply-design-system · /sync-design-tokens
UC 02

Generate Design

Produce actual screen designs from the system. Components, variables, and styles should be bound — but cleanup is part of the process.

/figma-generate-design
05 — Generate / Clean Up Design System

Figma Generate Library is the foundation

Foundation · 80% of the work
/figma-generate-library
Builds the entire library structure. Strict rules govern how it runs. Everything else depends on what this produces.
/audit-design-system
Checks component quality. Used only after foundations are established.
/apply-design-system
Applies and binds the system to components. Requires variables + styles to exist first.
/sync-design-tokens
Generates code schema from the Figma canvas. Useful backwards sync for our context.
Goal All bindings ready. Semantic meanings scoped. Designers can continue working entirely in Figma.
05.1 — Skill Deep Dive

/figma-generate-library workflow

This skill is never a one-shot task. It requires 20–100+ sequential Figma API calls across five mandatory phases. Skipping or reordering phases causes structural failures that are expensive to undo. Variables always come before components. Every phase ends with a required checkpoint.

01
Use it to start a design system from scratch — full build, nothing pre-existing.
02
Use it as a scoped implementation — run only the phases needed to set up a single component or a specific part of the system.
03
Phase 0 — Discovery — is always the first step. It cannot be skipped, even in a scoped run. No writes happen until Phase 0 is complete and approved.

07 — Patterns

Three working patterns

P1

Code-First

Figma as canvas. Design markdown + token schema are first-class. Figma DS is second-class.

P2

Figma as Design Tool

Designer-heavy. DS must be fully bound. Code generated from schema and Figma map pages.

P3

AI-First

Two sub-cases: exploration (fast, loose) and active production (strict, bound, markdown-driven).
