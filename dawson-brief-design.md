## Problem Statement

Portfolio managers at a PE firm (Dawson Partners demo context) spend 2-3 hours every
morning reading 10+ reports from scattered, non-centralized sources (PDFs, docs, PPTX)
before they can do their actual job. The consequence of missing something is a missed
trade action or an undetected risk event. There is no centralized intelligence layer
that understands their segment specifically.

## Demand Evidence

- 10 reports/day per PM, from external scattered websites — no internal platform
- Manual absorption every morning before any portfolio action can happen
- Consequence: missed trade or undetected risk (confirmed by client)
- Client requested this directly, motivated by maturing AI capability and efficiency
- Dawson Partners context: PE secondaries firm, preferred equity, LP liquidity —
  PMs monitor secondary deal flow, LP positions, preferred equity tranches

## Status Quo

Monday morning: PM opens 10+ reports across websites, reads PDFs/docs/PPTX manually,
cross-references against portfolio positions, checks news, monitors data changes.
No aggregation, no prioritization, no segment-aware filtering. Pure manual cognitive
load before any value-add work begins.

## Target User & Narrowest Wedge

**Target user:** Segment-specific portfolio manager at Dawson Partners.
**Demo shows exactly 2 personas** (3rd persona is out of scope for MVP):
- Persona 1 — Alex Chen, LP Secondaries PM: monitors NAV discounts, secondary market
  pricing, GP-led deal flow, continuation vehicle opportunities
- Persona 2 — Sarah Kim, Preferred Equity PM: tracks preferred return hurdles,
  PIK interest accrual, portfolio company coverage ratios, exit timing windows

A third persona (Structured Products) can be added if scope allows but is not required
for the demo to land.

**Wedge:** A single-page segment-aware morning brief. Opens at 8am, tells the PM
exactly what to pay attention to today, in their segment's language. Nothing else.

## Constraints

- This is a demo — data does not need to be real, it needs to feel real and
  Dawson-specific
- Segment personalization is non-negotiable from the first screen
- No live backend required — simulated intelligence is fine for demo
- Web-based dashboard, mobile-responsive

## Premises (agreed)

1. Core value is "time to context" — from zero to informed in minutes, not hours
2. Personalization by segment is critical from day one — different briefs for
   different PM types, not just filtered data
3. For the demo, data doesn't need to be real — it needs to look real and feel
   personalized
4. The brief must speak each segment's professional language (LP secondaries
   terminology for one PM, preferred equity language for another)

## Approaches Considered

### Approach A: Segment-Aware Morning Brief (CHOSEN)
Single-page daily brief per PM segment. AI-generated tone, segment-specific language.
Static/simulated data. Persona switcher in the UI to demonstrate personalization live
during the demo. All persona data pre-baked into the page — zero network requests on
persona switch, pure client-side DOM swap, <200ms transition.

Effort: M | Risk: Medium (domain language authenticity is required and must be validated)
Pros: Shippable fast, segment personalization is front and center, clean focused story
Cons: No real-time alerts, data is mocked

### Approach B: Brief + Live Alert Feed
Morning brief panel + simulated real-time alert stream with persona switching.
Effort: M | Risk: Medium
Pros: Live-updating "magic moment," covers both use cases
Cons: More moving parts, harder to keep demo stable

### Approach C: Conversational Brief
Chat-driven interface — PM asks "what should I watch today?" and gets segment-aware
AI response.
Effort: L | Risk: High
Pros: Futuristic feel
Cons: Requires real LLM backend to feel authentic, risks "just a chatbot" impression

## Recommended Approach

**Approach A: Segment-Aware Morning Brief**

One beautifully crafted brief per segment that speaks Dawson's language. The demo
story: switch between the LP Secondaries PM and the Preferred Equity PM — watch the
entire brief change. Different priorities, different risks, different language.
That persona switch IS the product demo.

Key UI elements:
- Segment persona selector (top of page, prominent) — switches full page content
- "Today's Brief" section: 3-5 bullet items in natural language, segment-aware
- "Key Risks" section: 2-3 flagged risks with severity indicator (High/Med)
- "Portfolio Snapshot": 4-6 key metrics as number cards (e.g. "NAV Discount: -12.4%")
- "What Changed Since Yesterday": 2-3 delta items (pre-baked "yesterday" values
  hardcoded alongside "today" values — no dynamic calculation required)
- Dawson logo (request from client) + dark financial aesthetic (#0D1117 background,
  neutral grays, single accent color for risk flags)

**Technical spec for persona switcher:**
Tech stack: vanilla HTML/CSS/JS (no framework — consistent with existing repo files).
All persona data lives in a single JS object at page load:
```js
const personas = {
  alex: {
    name: "Alex Chen", role: "LP Secondaries",
    brief: [...],         // array of {text: string, priority: "high"|"med"}
    risks: [...],         // array of {text: string, severity: "HIGH"|"MED"}
    snapshot: [...],      // array of {label: string, value: string, delta: string}
    deltas: [...]         // array of {metric: string, yesterday: string, today: string}
  },
  sarah: { ... }
}
```
Switching persona calls `renderPersona(personas[selected])` — pure DOM innerHTML
update, zero network, <200ms. Test on Chrome + Safari before demo day.

Today's Brief:

  Dawson Fund IV secondary position

Portfolio Snapshot:

What Changed Since Yesterday:
- Meridian NAV updated: -13.1% → -12.4% (+70bps improvement)

Use Dawson-authentic terminology:
- LP Secondaries: NAV discounts, secondary market pricing, liquidity events, GP-led
  transactions, continuation vehicles
- Preferred Equity: preferred return hurdles, PIK interest, coverage ratios,
  portfolio company performance, exit timing
- Structured Products: tranche performance, credit ratings, waterfall analysis,
  duration/convexity, spread movements
